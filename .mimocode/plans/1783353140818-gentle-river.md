# แผนงาน: ระบบ License Key + Subscription

## วัตถุประสงค์
สร้างระบบเช็คสิทธิ์การใช้งาน (License Key) สำหรับ Solar Dashboard — โปรแกรมต้องมี license key ที่ต่ออายุทุก 1 เดือน

---

## Phase 1: Database Design

### Table: `license`
```sql
CREATE TABLE IF NOT EXISTS license (
  id TEXT PRIMARY KEY,
  license_key TEXT NOT NULL UNIQUE,        -- key สำหรับ activate
  product_name TEXT DEFAULT 'Solar Dashboard',
  license_type TEXT DEFAULT 'subscription', -- subscription / perpetual
  activated_at DATETIME,                    -- วันที่ activate
  expires_at DATETIME NOT NULL,             -- วันหมดอายุ
  max_users INTEGER DEFAULT 10,             -- จำนวนผู้ใช้สูงสุด
  status TEXT DEFAULT 'active',             -- active / expired / suspended
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### Files ที่ต้องแก้ไข
- `backend/src/migrations/011_license_system.cjs` — สร้าง table
- `backend/src/routes/license.js` — API endpoints
- `backend/src/middleware/license.js` — License check middleware
- `backend/src/index.js` — เพิ่ม license route + middleware
- `electron/main.js` — เพิ่ม license check ก่อน start server
- `frontend/src/pages/LicenseSettings.jsx` — หน้าจัดการ license
- `frontend/src/components/LicenseBanner.jsx` — แสดง warning เมื่อใกล้หมดอายุ
- `frontend/src/context/AuthContext.jsx` — เก็บ license state
- `frontend/src/App.jsx` — เพิ่ม route สำหรับ license

## Phase 2: License Key Generation

### Algorithm
```
License Key Format: SOLAR-XXXX-XXXX-XXXX-XXXX
- Prefix: SOLAR
- 4 groups × 4 chars = 16 chars
- Characters: A-Z, 0-9 (ไม่ใช้ I, O, 1, 0 เพื่อป้องกันความสับสน)
- Checksum: last 4 chars = hash ของ first 12 chars
```

### ข้อมูลใน License Key
- **Product**: Solar Dashboard
- **Expiry**: embed ใน key (encrypted)
- **Max Users**: embed ใน key
- **License Type**: subscription (1 เดือน)

### การสร้าง Key
- Admin สร้างจากหน้า License Settings
- Key ถูก encode ด้วย simple encryption (AES-256)
- ข้อมูลถูก embed ใน key ไม่ต้อง online validate

## Phase 3: Backend API

### Endpoints

| Method | Path | หน้าที่ | Auth |
|--------|------|--------|------|
| `POST` | `/api/license/activate` | Activate ด้วย license key | Admin |
| `GET` | `/api/license` | ดูสถานะ license ปัจจุบัน | Admin |
| `GET` | `/api/license/validate` | ตรวจสอบว่า license ยังใช้ได้ | ทุกคน |
| `POST` | `/api/license/deactivate` | ปิดใช้งาน license | Admin |

## Phase 4: Electron Integration

### Startup Flow (ใหม่)
```
app.whenReady()
  → initializeDatabase()
  → checkLicense()          ← เพิ่มใหม่
  → ถ้า license ไม่ถูกต้อง → แสดง dialog "กรุณากรอก License Key"
  → ถ้า license หมดอายุ → แสดง dialog "License หมดอายุ กรุณาต่ออายุ"
  → ถ้า license ถูกต้อง → start Express server
```

### License Check Points
1. **Electron startup** — ก่อน start server
2. **Every API request** — middleware check
3. **Background check** — ทุก 1 ชั่วโมง (auto-lock ถ้า expire)

## Phase 5: Frontend Integration

### License Settings Page
- แสดงสถานะ license ปัจจุบัน (key, วันหมดอายุ, จำนวนผู้ใช้)
- ช่องกรอก license key สำหรับ activate
- ปุ่ม activate / deactivate
- แสดง warning เมื่อเหลือ < 7 วัน

### License Banner
- แสดง banner ด้านบนสุดของทุกหน้า
- สีเหลือง: เหลือ < 7 วัน
- สีแดง: หมดอายุแล้ว
- ซ่อน banner ถ้า license ยังปกติ

### Route Protection
- ถ้า license หมดอายุ → redirect ไปหน้า License Settings
- หน้า login ยังใช้ได้ (ต้อง login ก่อนถึงจะเห็น license)

---

## Phase 6: License Expiry Management

### Auto Lock
- Background job ตรวจสอบทุก 1 ชั่วโมง
- ถ้า expires_at < now → อัปเดต status = 'expired'
- ถ้า status = 'expired' → block ทุก API request (ยกเว้น login + license)

### Notification
- 7 วันก่อนหมดอายุ → แสดง warning banner
- 1 วันก่อนหมดอายุ → แสดง warning สีแดง
- วันหมดอายุ → แสดง dialog "License หมดอายุ"

---

## Phase 7: Security Considerations

### License Key Protection
- Key ถูก encode ด้วย AES-256
- ข้อมูลถูก embed ใน key (ไม่เก็บ key แบบ plain text ใน DB)
- ตรวจสอบ checksum เพื่อป้องกัน key ปลอม

### Anti-tampering
- ตรวจสอบ hardware fingerprint (optional)
- ตรวจสอบ DB integrity
- ตรวจสอบ file modification
