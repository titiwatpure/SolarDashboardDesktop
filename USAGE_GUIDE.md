## การเริ่มต้น

### ตรวจสอบ Node.js
```bash
node --version   # ควรเป็น v18 ขึ้นไป
npm --version
```

### ติดตั้ง Backend
```bash
cd backend
npm install
cp .env.example .env
node src/init-db.cjs
npm run dev
```

### ติดตั้ง Frontend (เปิด terminal ใหม่)
```bash
cd frontend
npm install
cp .env.example .env
npm start
```

### หรือใช้ Docker
```bash
docker compose up -d --build
```

## การเข้าสู่ระบบ

เปิด http://localhost:3000

| Username | Password | บทบาท |
|----------|----------|-------|
| admin | admin | Admin |
| engineer | engineer | Engineer |
| staff | staff | Staff |
| client | client | Client |

### บทบาทผู้ใช้
- **Admin** - สิทธิ์เต็ม: CRUD ทุกอย่าง, จัดการผู้ใช้, ดู activity logs, backup/restore
- **Engineer** - ดูข้อมูล, สร้าง/แก้ไขโครงการ, อัปโหลดเอกสาร, จัดการ checkpoint
- **Staff** - ดูข้อมูล, สร้าง/แก้ไขโครงการบางส่วน, อัปโหลดเอกสาร
- **Client** - ดูข้อมูลอย่างเดียว (อ่าน)

### สิทธิ์ละเอียด (Granular Permissions)
นอกจากบทบาทแล้ว ยังสามารถกำหนดสิทธิ์ละเอียดได้:
- `project.create` - สร้างโครงการ
- `project.update.all` - แก้ไขโครงการทั้งหมด
- `project.delete` - ลบโครงการ
- `approval.manage` - จัดการอนุมัติหน่วยงาน
- `checkpoint.approve` - อนุมัติ Checkpoint

## การใช้งานแต่ละหน้า

### หน้าแรก (Dashboard)
- **KPI Cards**: จำนวนโครงการทั้งหมด, ประเภทใบอนุญาต, เสร็จ, ติดปัญหา, ความเสี่ยงสูง/วิกฤต
- **Pipeline**: แสดงขั้นตอน 7 ขั้น + สถานะแยกตามสี
- **Pie Chart**: สัดส่วนสถานะโครงการโดยรวม

### โครงการ (Projects)
- รายการโครงการทั้งหมดพร้อม filter (จังหวัด, สถานะ, ขั้นตอน, ความเสี่ยง)
- Pagination + ค้นหา
- แสดงคอลัมน์: ลูกค้า, หน่วยงาน, ผู้รับผิดชอบ
- คลิกชื่อโครงการเพื่อดูรายละเอียด
- คลิก "ดูรายงาน" เพื่อดูรายงานรายโครงการ
- **เพิ่มโครงการ**: กรอกข้อมูลพื้นฐาน + เลือกลูกค้า + ระบุสถานที่/สัญญา (ทุก field ไม่บังคับ)

### รายละเอียดโครงการ (ProjectDetail)
- ข้อมูลโครงการ + ผู้รับผิดชอบ + Progress Bar
- **ข้อมูลลูกค้า**: แสดงข้อมูลลูกค้าที่เชื่อม (ถ้ามี)
- **สถานที่ติดตั้ง**: ที่อยู่, พิกัด, สถานีไฟฟ้า (ถ้ามี)
- **สัญญา/การเงิน**: เลขที่สัญญา, มูลค่า, งบประมาณ (ถ้ามี)
- **สเปคเทคนิค**: แผง, อินเวอร์เตอร์, ประเภทติดตั้ง (ถ้ามี)
- **Pipeline Steps**: แสดง 7 ขั้นตอน + Checkpoint แต่ละขั้น
- **จัดการ Checkpoint**: อนุมัติ/ไม่ผ่าน/ข้าม + ดูประวัติ
- **Timeline + ความคิดเห็น**: บันทึกการเปลี่ยนแปลงอัตโนมัติ + แสดงความคิดเห็น
- **จัดการหน่วยงาน**: เพิ่ม/อนุมัติ/ปฏิเสธหน่วยงานที่เกี่ยวข้อง
- **เลือกผู้รับผิดชอบ**: เปลี่ยนผู้รับผิดชอบโครงการ

### รายงานรายโครงการ (ProjectReport)
- รายงานละเอียดของแต่ละโครงการ
- แสดง: ขั้นตอน, Checkpoint, งาน, เอกสาร, หน่วยงาน, ความเสี่ยง, Timeline
- Export PDF + Print
- เข้าจาก: คลิก "ดูรายงาน" ในตารางโครงการ

### ขั้นตอน (Steps)
- Pipeline Visualization แสดงแต่ละขั้นตอนเป็นวงกลม
- แสดงจำนวนโครงการ + สถานะแยกตามขั้นตอน

### งาน (Tasks)
- รายการงานทั้งหมด + กรองตามสถานะ/Priority/โครงการ
- สร้าง/แก้ไข/ลบงาน
- Priority: ต่ำ, ปานกลาง, สูง, เร่งด่วน
- สถานะ: รอดำเนินการ, กำลังทำ, เสร็จแล้ว, ยกเลิก
- มอบหมายงานให้ผู้ใช้ + แจ้งเตือนอัตโนมัติ
- ไฮไลท์งานเกินกำหนด (Overdue)

### เอกสาร (Documents)
- **อัปโหลดไฟล์จริง** รองรับ PDF, Word, Excel, PowerPoint, รูปภาพ, ZIP/RAR (สูงสุด 50MB)
- ลากไฟล์มาวาง (drag-and-drop) หรือเลือกไฟล์
- ดาวน์โหลดไฟล์จากตาราง
- สรุปจำนวนเอกสารตามประเภท

### รายงาน (Reports)
- **12 รายงาน** พร้อมเลือกแสดง/ซ่อนแต่ละส่วน:
  1. สรุปตามสถานะ (Pie Chart)
  2. สรุปตามขนาด (Bar Chart)
  3. สรุปตามจังหวัด (ตาราง + % เสร็จสิ้น)
  4. สรุปตามขั้นตอน (Horizontal Bar)
  5. สรุปขั้นตอน + สถานะ (Pipeline)
  6. ประวัติความคืบหน้า (ตาราง + Pagination + ค้นหา)
  7. รายงานความเสี่ยง
  8. ระยะเวลาแต่ละขั้นตอน
  9. ผลงานรายบุคคล
  10. สรุปงานตาม Priority
  11. **สรุปงานตามผู้รับผิดชอบ** — แสดงชื่อผู้รับผิดชอบ + จำนวนงานแต่ละสถานะ + งานเกินกำหนด
  12. **รายละเอียดงานทั้งหมด** — รายการงานทั้งหมดพร้อมชื่อผู้รับผิดชอบ, โครงการ, Priority
- Export Excel (XLSX) / PDF รองรับภาษาไทย (ฟอนต์ Sarabun)
- Excel ส่งออกหลาย Sheet: "สรุปตามผู้รับผิดชอบ" + "รายละเอียดงาน"

### ลูกค้า (Customers)
- รายการลูกค้าทั้งหมด + ค้นหา
- เพิ่ม/แก้ไข/ลบลูกค้า
- ข้อมูล: ชื่อ, ประเภท (บุคคล/นิติบุคคล/ราชการ), ผู้ติดต่อ, เบอร์โทร, อีเมล, เลขภาษี, ที่อยู่
- แสดงจำนวนโครงการที่เชื่อม
- สร้างลูกค้าใหม่จากหน้าเพิ่มโครงการได้เลย

### หน่วยงาน (Organizations)
- รายการหน่วยงาน (กกพ., PEA, MEA, ฯลฯ)
- ดูโครงการที่เชื่อมโยง
- เพิ่ม/แก้ไข/ลบหน่วยงาน
- ระบบอนุมัติ: pending -> approved / rejected (พร้อมเหตุผล)

### ผู้ใช้งาน (Users) - Admin only
- รายการผู้ใช้ทั้งหมด
- เพิ่ม/แก้ไข/ลบผู้ใช้
- กำหนดบทบาท + สิทธิ์ละเอียด

### ตั้งค่า (Settings)
- ดูข้อมูลผู้ใช้ปัจจุบัน
- เปลี่ยนรหัสผ่าน (อย่างน้อย 8 ตัวอักษร)
- ตั้งค่าข้อมูลบริษัท (ชื่อ, ที่อยู่, เลขภาษี, เบอร์โทร, อีเมล)
- **อัปโหลดโลโก้บริษัท**: เลือกไฟล์รูปภาพ → แสดง preview → กดอัปโหลด → แสดงใน Dashboard และ PDF reports
- **ตั้งค่าทั่วไป**: ภาษา (ไทย/อังกฤษ), ธีม, รูปแบบวันที่, เขตเวลา (มีผลทั้งระบบ)
- **ประวัติเวอร์ชัน**: แสดง timeline การอัปเดตแต่ละเวอร์ชัน

### สัญญา (Contracts)
- รายการสัญญาทั้งหมด
- สร้าง/แก้ไข/ลบสัญญา
- เชื่อมกับโครงการและลูกค้า
- สถานะ: ร่าง, มีผล, เสร็จสิ้น, ยกเลิก

### บัญชี (Accounting)
- 3 แท็บ: สรุปโครงการ, ธุรกรรม, งวดชำระ
- บันทึกรายรับ-รายจ่าย + ผูกกับโครงการ
- สร้างงวดชำระจากสัญญา + ชำระเต็ม/บางส่วน
- สรุปการเงิน: รายรับ, รายจ่าย, กำไร, ยอดค้างชำระ

### พอร์ทัลลูกค้า (Portal)
- หน้าสำหรับลูกค้าดูข้อมูลตัวเอง
- ดูโครงการ, ใบเสนอราคา, สัญญา, เอกสาร

### แผนที่ (Network Map)
- แสดงตำแหน่งโครงการบนแผนที่
- ใช้พิกัด lat/lng จากข้อมูลโครงการ

## การจัดการลูกค้า

### เพิ่มลูกค้า
1. ไปหน้า "ลูกค้า" จาก Sidebar
2. กดปุ่ม "เพิ่มลูกค้า"
3. กรอกข้อมูล (ทุก field ไม่บังคับ ยกเว้นชื่อลูกค้า):
   - ชื่อลูกค้า (จำเป็น)
   - ประเภท: บุคคลธรรมดา / นิติบุคคล / หน่วยงานราชการ
   - ผู้ติดต่อ, เบอร์โทร, อีเมล
   - เลขประจำตัวผู้เสียภาษี
   - ที่อยู่, หมายเหตุ
4. กด "บันทึก"

### สร้างลูกค้าจากหน้าเพิ่มโครงการ
1. เปิด modal เพิ่ม/แก้ไขโครงการ
2. ส่วน "ข้อมูลลูกค้า" กด "+ สร้างใหม่"
3. กรอกข้อมูลลูกค้าขั้นต่ำ → กด "สร้างลูกค้า"
4. ลูกค้าจะถูกเลือกอัตโนมัติ

### ผูกลูกค้ากับโครงการ
- เลือกลูกค้าจาก dropdown ใน modal แก้ไขโครงการ
- 1 ลูกค้ามีได้หลายโครงการ

## การอัปเดตสถานะโครงการ

### วิธีที่ 1: จากตาราง
1. ไปหน้า Dashboard หรือ Projects
2. หาโครงการ -> คลิกปุ่ม "อัปเดตสถานะ" (ไอคอนธง)
3. เลือกขั้นตอนและสถานะ
4. บันทึก

### วิธีที่ 2: จากหน้ารายละเอียด
1. คลิกชื่อโครงการ
2. ดู Progress Bar 7 ขั้นตอน
3. อัปเดตจากขั้นตอนที่ active

### ขั้นตอนทั้ง 7
```
1. Survey (สำรวจพื้นที่)
2. Design (ออกแบบ)
3. ERC (ขอใบอนุญาต กกพ.)
4. Grid (ขอเชื่อมต่อไฟฟ้า)
5. Construction (ก่อสร้าง)
6. Testing (ทดสอบระบบ)
7. COD (จ่ายไฟเชิงพาณิชย์) -> สถานะ "เสร็จสิ้น" อัตโนมัติ
```

### สถานะ
- **ยังไม่เริ่ม** - ยังไม่เริ่มงาน
- **กำลังดำเนินการ** - ทำงานอยู่
- **รอข้อมูล** - รอการตรวจสอบหรืออนุมัติ
- **ติดปัญหา** - ต้องระบุเหตุผล
- **ถูกปฏิเสธ** - ถูกปฏิเสธและต้องแก้ไข
- **เสร็จสิ้น** - งานเสร็จ

### สิ่งที่ระบบทำอัตโนมัติ
- บันทึก Timeline ทุกครั้งที่เปลี่ยนสถานะ
- ส่ง Notification ให้ผู้รับผิดชอบ
- บันทึก Activity Log
- ถ้าขั้นตอนเป็น COD -> ล็อกสถานะเป็น "เสร็จสิ้น"
- สร้าง Checkpoint อัตโนมัติเมื่อเข้าขั้นตอนใหม่
- คำนวณความเสี่ยงอัตโนมัติ

## ระบบ Checkpoint

### Checkpoint คืออะไร
จุดตรวจสอบในแต่ละขั้นตอนของโครงการ เช่น ขั้นตอน Survey มี 3 จุดตรวจสอบ:
- สำรวจพื้นที่
- ตรวจสอบโครงสร้างหลังคา
- ประเมินระบบไฟฟ้า

### การใช้งาน
1. ไปหน้ารายละเอียดโครงการ
2. แต่ละขั้นตอนจะแสดง Checkpoint ที่เกี่ยวข้อง
3. กดปุ่มเพื่อ:
   - **ผ่าน** (✓) - Checkpoint ผ่าน
   - **ไม่ผ่าน** (✗) - Checkpoint ไม่ผ่าน (ระบุเหตุผล)
   - **ข้าม** - ข้าม Checkpoint นี้
4. ดูประวัติการเปลี่ยนแปลงได้

### สิ่งที่ระบบทำอัตโนมัติ
- สร้าง Checkpoint อัตโนมัติเมื่อเข้าขั้นตอนใหม่
- แจ้งเตือนเมื่อ Checkpoint ไม่ผ่าน
- บันทึกประวัติการเปลี่ยนแปลง
- คำนวณความเสี่ยงใหม่เมื่อสถานะ Checkpoint เปลี่ยน

## ระบบความเสี่ยง

### การคำนวณ
ระบบคำนวณคะแนนความเสี่ยงอัตโนมัติจาก 5 ปัจจัย:

| ปัจจัย | คะแนนสูงสุด | เงื่อนไข |
|--------|-------------|---------|
| ความล่าช้า | 40 | เปรียบเทียบเวลากำหนด COD |
| การติดปัญหา | 40 | จำนวนวันที่ blocked |
| Checkpoint ไม่ผ่าน | 30 | จำนวน Checkpoint ที่ failed |
| Task เกินกำหนด | 20 | จำนวนงาน overdue |
| สถานะถูกปฏิเสธ | 20 | สถานะ rejected |

### ระดับความเสี่ยง
| ระดับ | คะแนน | สี |
|-------|-------|---|
| ต่ำ | < 25 | เขียว |
| ปานกลาง | 25-49 | เหลือง |
| สูง | 50-79 | ส้ม |
| วิกฤต | 80+ | แดง |

### เมื่อไหร่ที่คำนวณใหม่
- อัปเดตสถานะโครงการ
- เปลี่ยนขั้นตอน
- อัปเดตสถานะ Checkpoint
- อัปเดตงาน

## การจัดการเอกสาร

### อัปโหลดไฟล์
1. ไปหน้า Documents
2. เลือกโครงการ
3. ตั้งชื่อเอกสาร (หรือปล่อยว่างให้ใช้ชื่อไฟล์)
4. เลือกประเภทเอกสาร
5. **ลากไฟล์มาวาง** หรือคลิก "เลือกไฟล์"
6. กด "บันทึกเอกสาร"

## ระบบบัญชี

### หน้าบัญชีมี 3 แท็บ

#### 1. สรุปภาพรวม (Overview)
- สรุปการเงินทั้งบริษัท: รายรับ, รายจ่าย, กำไร
- แนวโน้มรายเดือน (12 เดือนล่าสุด)
- โครงการที่ทำกำไรมากที่สุด 5 อันดับ
- ลูกหนี้คงค้าง (งวดที่ยังไม่ชำระ)
- **อัปเดตอัตโนมัติ** เมื่อเพิ่ม/แก้ไข/ลบรายการบัญชีหรืองวดชำระ

#### 2. โครงการ (Project Summary)
- สรุปการเงินรายโครงการ: รายรับ, รายจ่าย, กำไร, ยอดค้างชำระ
- เลือกโครงการจาก dropdown
- **อัปเดตอัตโนมัติ** เมื่อเพิ่ม/แก้ไข/ลบรายการบัญชีหรืองวดชำระของโครงการนั้น

#### 3. ธุรกรรม (Transactions)
- บันทึกรายรับ-รายจ่าย
- ผูกกับโครงการ (ไม่บังคับ) + หมวดหมู่
- กรองตามวันที่, โครงการ, หมวดหมู่, ประเภท (รายรับ/รายจ่าย)
- สร้าง/แก้ไข/ลบ

#### 4. งวดชำระ (Installments)
- **สร้างงวดชำระจากสัญญา**: เลือกสัญญา → กำหนดงวด (เช่น 30-30-40 หรือ 50-50)
- **ชำระเงิน**: เลือกงวด → ชำระเต็มหรือบางส่วน
  - ชำระเต็ม → สถานะ "ชำระแล้ว" + สร้าง transaction อัตโนมัติ
  - ชำระบางส่วน → สถานะ "ชำระบางส่วน" + สร้าง transaction
- **ลบงวด**:
  - ปกติ: กดลบ → ยืนยัน → ลบสำเร็จ
  - ถ้างวดมีรายการบัญชีเชื่อมอยู่: ระบบแสดงข้อมูลรายการบัญชี (ประเภท/ยอดเงิน) → กด "ตกลง" เพื่อลบทั้งงวดชำระและรายการบัญชีพร้อมกัน (Force Delete)
- สถานะงวด: รอชำระ, ชำระบางส่วน, ชำระแล้ว, เกินกำหนด
- วันครบกำหนดไม่บังคับ (แสดง "ยังไม่กำหนด" ถ้าไม่ได้กรอก)

> **หมายเหตุ**:
> - ตัวเลขสรุปภาพรวม (Overview) และสรุปโครงการ จะอัปเดตทันทีหลังเพิ่ม/แก้ไข/ลบรายการบัญชีหรืองวดชำระ ไม่ต้องรีเฟรชหน้า
> - เมื่อลบโครงการ รายการบัญชีและงวดชำระที่เกี่ยวข้องจะถูกลบอัตโนมัติ (CASCADE delete)
> - สรุปบัญชีจะแสดงเฉพาะข้อมูลที่ยังมีโครงการอยู่เท่านั้น (ไม่แสดงข้อมูลโครงการที่ถูกลบแล้ว)

## การติดตั้งในองค์กร

### ตัวเลือก 1: รันบนเครื่องเดียว (LAN)
1. ติดตั้ง Node.js 18 บนเครื่อง server
2. copy โปรเจกต์ไป
3. รัน `npm install` + `node src/init-db.cjs`
4. แก้ CORS ให้รับทุก IP:
   ```
   # backend/.env
   CORS_ORIGIN=*
   ```
5. รัน backend: `npm start`
6. build frontend: `cd frontend && npm run build`
7. เปิด browser เข้า `http://IPเครื่อง:5000`

### ตัวเลือก 2: Docker
1. ติดตั้ง Docker Desktop
2. copy โปรเจกต์ไป
3. รัน `docker compose up -d --build`
4. เปิด `http://IPเครื่อง:3000`

### ตัวเลือก 3: Render.com
1. Push โค้ดขึ้น GitHub
2. สร้าง Web Service บน Render (Backend)
3. สร้าง Static Site บน Render (Frontend)
4. ตั้ง Environment Variables ตาม `.env.example`

### Backup
```bash
# ข้อมูลทั้งหมดอยู่ในไฟล์เดียว
cp backend/solar_dashboard.db backup_$(date +%Y%m%d).db
```

### Restore
```bash
cp backup_20260101.db backend/solar_dashboard.db
```

## การอัปเดตแอป

```bash
# 1. Backup database
cp backend/solar_dashboard.db backup.db

# 2. ดึงโค้ดใหม่
git pull

# 3. ลง dependencies
cd backend && npm install
cd ../frontend && npm install

# 4. รัน init-db (ถ้ามีตารางใหม่)
cd ../backend && node src/init-db.cjs

# 5. รันใหม่
npm run dev
```

> ข้อมูลเดิมไม่หาย (CREATE TABLE IF NOT EXISTS + INSERT OR IGNORE)

## แก้ปัญหาเบื้องต้น

| ปัญหา | วิธีแก้ |
|-------|---------|
| เปิดหน้าเว็บไม่ได้ | ตรวจสอบ backend รันอยู่หรือไม่ |
| Login ไม่ได้ | ตรวจสอบ username/password (admin/admin) |
| CORS error | แก้ `CORS_ORIGIN` ใน `backend/.env` |
| Upload ไฟล์ไม่ได้ | ตรวจสอบขนาดไฟล์ (สูงสุด 50MB) |
| Token หมดอายุ | Login ใหม่ (ระบบมี refresh token อัตโนมัติ) |
| ข้อมูลหาย | ตรวจสอบ `solar_dashboard.db` ยังอยู่หรือไม่ |
| ข้อมูลหายบน Render | ใช้ Persistent Disk หรือ PostgreSQL |
| Port ถูกใช้ | เปลี่ยน `PORT` ใน `.env` |
